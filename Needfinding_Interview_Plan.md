# Needfinding Plan — Semi-structured interview & think-aloud (Activity 1 of 2)

**Task domain:** Improving how students interpret **AP Classroom** scores and feedback for **practice and assignments** (e.g., progress checks, topic quizzes, **and** written/FRQ-style work—not limited to FRQs only).

**Method:** Semi-structured interview with a short **think-aloud** on the participant’s **real** feedback page in AP Classroom.

**Target sample:** **n = 5** participants · **~18–22 minutes** each (aligns with course guidance for interviews).

**Companion activity (separate write-up):** A **heuristic evaluation** of the **current** AP Classroom student feedback interface (≥3 heuristics). Document that in the body under *Needfinding Plan* / *Results*; it is **not** duplicated in this script.

## 2. Ethics, privacy, and consent (read / confirm at session start)

- **Purpose:** Course project to understand how students read feedback; **not** a test of the participant’s ability.
- **Data:** Notes and **short quotes** only. **Do not** copy or photograph the participant’s **full submission text** (code or written answer). You may note **general** observations (e.g., “comment only said ‘see rubric’”).
- **Say aloud:**  
  *“I won’t copy your full submission. I’ll only write general observations and short quotes about how you experience the feedback page.”*
- **Consent:**  
  - OK to take notes?  
  - (Optional) OK to **audio record** for transcription? If no, rely on notes only.

---

## 3. Materials you need

- Link or path to **AP Classroom** (participant uses **their** account).
- Video call with **screen share**, or in-person where they can show the screen.
- Note template or Notion database (see §8) with **Participant ID** P1–P5.

---

## 4. Session flow (print-friendly)

| Phase | Time | What you do |
| --- | --- | --- |
| Opening + consent | ~1 min | Thank them; purpose; privacy; consent for notes/recording |
| Open feedback page | ~1–2 min | They open **the most recent assignment or practice** they have feedback for (any type). If unavailable: *“Any feedback page you remember clearly is fine.”* |
| Think-aloud (Task 1) | ~6–8 min | They **think aloud** while walking the page top to bottom |
| Structured questions | ~5–7 min | Questions below, in order |
| Follow-up probes | ~3–5 min | Use neutral probes only if stuck |
| Closing | ~30 sec | One final reflective question |
| Post-session | ~5 min (yours) | Fill synthesis row in database + jot 1-sentence takeaway |

**Think-aloud instruction (read aloud):**

> *“Please think aloud: say what you’re looking at, what you think it means, and what you would do next. I’ll take notes about the interface and your experience—not about your grade.”*

---

## 5. Task 1 — Understand the feedback (during think-aloud)

Ask **in order** (let them point/share screen):

1. **Where** do you believe you lost points (or missed credit)? — *“Point to what on the page tells you that.”*
2. **Why** do you think those points were deducted? — *“In your own words—what is the system saying you missed?”*
3. **Which rubric row, skill tag, or criterion** does this relate to (if shown)? — *“Where would you look if it’s not obvious?”*
4. **What would you do next** to improve? — *“One concrete next step.”*
5. **Confidence:** On a scale of **1–5**, how confident are you that you could improve next time **based only on this feedback**?

---

## 6. Success criteria (what “good” needfinding looks like for you as researcher)

During Task 1, check whether the participant can (even partially):

- [ ] **Locate** where the deduction or issue is indicated  
- [ ] **Explain why** in their own words  
- [ ] **Relate** the feedback to a criterion / rubric / skill (or say where they would find it)  
- [ ] State **one concrete next step**  
- [ ] Give a **confidence** rating (1–5)

---

## 7. Neutral probes (use only if stuck)

- What are you looking for right now?  
- What does this wording mean to you?  
- What information would you want to see here?  
- If you could change **one thing** on this page, what would it be?

---

## 8. Follow-up questions (after Task 1)

1. What was the **most helpful** part of this feedback? Why?  
2. What was the **least helpful or most confusing** part? Why?  
3. What is **missing** that you would need to revise confidently? *(e.g., rubric mapping, evidence tied to your answer, step-by-step fix, practice suggestions)*  
4. Do you **trust** this feedback? What would **increase** trust?  
5. If you could add **only one** improvement to this experience, what would it be?

**Closing:**

> *“Thanks. Last question: after reading this feedback, do you feel you know **exactly** what to do next? Why or why not?”*

Create a database, e.g. **`Practice Feedback Needfinding`**, with properties such as:

| Property | Type | Notes |
| --- | --- | --- |
| **Participant ID** | Title | P1 … P5 |
| **Course** | Select | CSA / CSP / Other AP |
| **Assignment type** | Select | Progress check / Quiz / Written or FRQ-style / Other |
| **Topic or unit** | Text | Short label (e.g., “Unit 4 loops”) |
| **Locate deduction** | Checkbox | |
| **Explain why** | Checkbox | |
| **Map to rubric/criterion** | Select | Yes / No / Not visible |
| **Concrete next step** | Checkbox | |
| **Confidence (1–5)** | Number | |
| **Biggest confusion** | Text | Short phrase |
| **Missing info** | Multi-select | Examples / Rubric link / Highlighted evidence / Step-by-step plan / Practice suggestion / Other |
| **Trust concerns** | Text | Optional |
| **Best quote** | Text | Short, de-identified |
| **Top improvement request** | Text | |
| **Themes / tags** | Multi-select | Vague feedback / Rubric gap / Not actionable / Needs examples / Trust / Next-step guidance / … |

**Useful views:** filter **Confidence ≤ 3**; group by **Missing info** for synthesis.
