# Individual Project To-Do

**与课程终稿要求对照**（篇幅、章节、样本量、风险点）：见 [`Individual_Project_Assignment_Requirements.md`](Individual_Project_Assignment_Requirements.md) 全文，尤其 **§5 方案检查** 与 **§7 章节—文件对照**。

## Priority Order

1. Finish interview-note整理 and write `Needfinding Results`
2. Add a formal `Heuristic Evaluation`
3. Turn the low-fi paper survey (`n = 4`) into a clean `Evaluation Results` section
4. Run and write the `Final Prototype Evaluation`
5. Organize appendices and supporting artifacts

## 1. Needfinding

- [ ] Target **n = 5** interviews for needfinding (see `Needfinding_Interview_Plan.md`); confirm final count in Results
- [ ] Confirm interview count: write down exactly how many interview recordings you have
- [ ] For each interview, create a short summary table with:
  - participant ID
  - background / FRQ experience
  - duration
  - main pain points
  - useful quote or paraphrase
  - design implication
- [ ] Write a short description of recruitment:
  - who participants were
  - how you recruited them
  - why they fit the task
- [ ] Extract at least 3 clear needfinding insights from interviews
- [ ] Make sure these insights are evidence-based, not only personal opinion
- [ ] Decide whether classroom observation / teaching experience will be included as contextual support

## 2. Heuristic Evaluation

- [ ] Pick the current interface to evaluate:
  - AP Classroom AP CSA FRQ student-side feedback / score report
- [ ] Choose at least 3 heuristics
  - Visibility of system status
  - Match between system and users' real goals
  - Recognition rather than recall
  - Help users recognize and recover from errors
- [ ] For each heuristic, write:
  - what part of the current interface supports or violates it
  - why that matters for students
  - how it connects to your redesign direction
- [ ] Summarize the heuristic evaluation findings in a clean table or short subsection

## 3. Needfinding Results Section

- [ ] Write what you planned to do
- [ ] Write what you actually did
- [ ] Report actual participant count and duration
- [ ] Summarize interview findings
- [ ] Summarize heuristic evaluation findings
- [ ] End with at least 3 insights

## 4. Low-Fi Prototype Evaluation (`A/B/C`, `n = 4`)

- [ ] Treat this as `Evaluation Results`, not as early needfinding
- [ ] Create a raw data table for all 4 participants
- [ ] Code the survey data:
  - `SD = 1`
  - `D = 2`
  - `A = 3`
  - `SA = 4`
- [ ] Record:
  - FRQ experience
  - pre-confidence
  - A1-A5
  - B1-B5
  - C1-C5
  - default choice
  - rank A/B/C
  - post-confidence
  - open-ended responses
- [ ] Compute summary statistics:
  - mean for each item
  - overall mean for A / B / C
  - times chosen as default
  - average rank
- [ ] Write a short limitation note:
  - sample was small (`n = 4`)
  - results are formative and trend-based, not strong statistical claims
- [ ] Extract 3-4 qualitative themes from open-ended responses
- [ ] Use these results to justify why you moved toward the final design direction

## 5. Evaluation Planning Section

- [ ] Rewrite your first-round evaluation plan in a more formal way
- [ ] Include:
  - participants
  - recruitment
  - procedure
  - quantitative measures
  - qualitative measures
  - analysis plan
- [ ] Make sure the plan matches the low-fi paper survey you actually used

## 6. Evaluation Results Section

- [ ] Report actual execution:
  - how many participants
  - where / how the survey happened
  - what materials they saw
- [ ] Report quantitative results
- [ ] Report qualitative themes
- [ ] Interpret what these findings mean for iteration

## 7. Second Iteration Planning

- [ ] Explicitly explain why the first-round results led to the final design
- [ ] State which parts came from:
  - Design A
  - Design B
  - Design C
- [ ] State what problems still remained after the first iteration
- [ ] Explain what the final prototype was intended to solve better

## 8. Final Prototype

- [ ] Capture multiple screenshots of the medium-fidelity prototype
- [ ] Write rationale for key design choices
- [ ] Tie each major design decision back to:
  - interview findings
  - heuristic evaluation
  - low-fi evaluation results

## 9. Video Prototype

- [ ] Confirm there is a working video link
- [ ] Confirm access permissions work
- [ ] Add short instructions for viewing
- [ ] Mention what the video demonstrates

## 10. Final Evaluation

- [ ] Run a new evaluation for the final prototype
- [ ] Do not reuse only the low-fi `A/B/C` data for this section
- [ ] Recommended format:
  - participants view the video prototype or explore the live demo
  - participants complete a short questionnaire
- [ ] Recommended sample target:
  - survey: ideally `8-15`, better if closer to `20`
  - interview / think-aloud: about `5`
- [ ] Use both quantitative and qualitative questions

## 11. Final Evaluation Planning Section

- [ ] Write:
  - who the participants are
  - how they will be recruited
  - what they will do
  - how long it takes
- [ ] Include quantitative items such as:
  - understanding why points were lost
  - identifying where the issue occurred
  - knowing what to practice next
  - clarity of feedback-to-action flow
  - willingness to use again
- [ ] Include qualitative items such as:
  - most useful part
  - confusing part
  - what is missing
  - suggested improvement
- [ ] Include analysis plan:
  - descriptive statistics for quantitative responses
  - thematic coding / summary for qualitative responses

## 12. Final Evaluation Results Section

- [ ] Report actual participant count
- [ ] Report participant background
- [ ] Report whether they watched video / used demo
- [ ] Compute quantitative summaries
- [ ] Summarize 3-4 qualitative themes
- [ ] Interpret whether the final prototype met the project goals
- [ ] Mention remaining weaknesses or next steps

## 13. Appendices

- [ ] Interview script or interview prompts
- [ ] Interview notes / summaries
- [ ] Heuristic evaluation notes
- [ ] Brainstorming notes
- [ ] AI brainstorming transcript if used
- [ ] Full low-fi wireframes
- [ ] Paper survey instrument
- [ ] Photos / scans of raw low-fi survey responses
- [ ] Typed raw data table from low-fi survey
- [ ] Final prototype screenshots
- [ ] Final evaluation questionnaire
- [ ] Final evaluation raw results
- [ ] Video prototype link / access note

## 14. Writing Checks

- [ ] Keep the task scope narrow and concrete
- [ ] Make sure each section matches the rubric heading
- [ ] Do not confuse:
  - needfinding
  - first prototype evaluation
  - final prototype evaluation
- [ ] Keep the main body under 25 pages
- [ ] Put bulky raw materials in appendices
- [ ] Make every claim traceable to data or observation

## Suggested Minimum Data Package

- Interviews: ideally `4-5`
- Low-fi prototype comparison: current `n = 4` is usable
- Final prototype evaluation:
  - survey: preferably `8+`
  - or interview / think-aloud: about `5`
