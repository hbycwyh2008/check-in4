# Individual Project — 作业要求整理（课程说明摘要）

> 来源：你提供的课程 Individual Project 说明与 Canvas 评分结构。正式提交以 Canvas / 官方 PDF 为准。  
> **篇幅**：正文最多 **25 页**（不含参考文献与附录）；25 为上限非目标。需 **JDF** 排版，导出 **单份 PDF** 提交。附录用于粘贴问卷全文、访谈提纲、原始笔记、完整原型图、评估原始数据等，**一般不视为额外大量新写作**。

---

## 1. 项目总览（类比）

- 个人项目 ≈ 在公司里为争取资源做的 **内部 pitch**；团队项目 ≈ 公司立项后 **正式组队推进**（规模更大）。
- **团队项目不得延续** 任何组员个人项目的同一选题；可同一大领域但需 **足够不同**（个人项目的原型不能直接当团队项目原型）。

### 选题建议（非强制规则）

- 受众面 **够大**，便于招募。
- 受众 **你够得着**（同学学分参与也可，但圈外亲友往往更省事）。
- 强调 **具体任务（task）**，不要整站/整 App 重做。
- 任务 **常见**，便于互评与助教理解。
- **个人项目**：必须针对 **已有界面** 做改进；**团队项目** 才可做尚无界面的任务。

### Check-in

- 共 **4 次**；按 **完成度** 记，**内容不单独给分**。
- 缺交：从项目总分扣 **每次 5 分**（课程说明原文）。

## 3. 各章节要点（与评分对应）

### Introduction（~1 页）

- 任务是什么、用户要完成什么；可含 **故事/小场景** 说明为何关心该领域。

### Needfinding Plan（~1 页 + 附录）

至少需要 **两类** needfinding：

1. **与用户主动互动**（问卷 / 访谈 / 出声思考 / 学徒观察等）。  
   - 问卷：约 **~20 人** × **~10 分钟** 量级。  
   - 访谈：约 **~5 人** × **~20 分钟** 量级。  
   - 学徒式：**1–2 人** × **约 1 小时** 量级。  
   - 正文写清：**谁、如何招募、做什么**；完整题干可放附录。

2. **启发式评估（Heuristic evaluation）**  
   - 针对 **现有界面**（与你任务相关）。  
   - 至少 **3 条** 启发式（可用 Nielsen、课程 2.5 十五条、或基于执行/评估鸿沟等自拟）。  
   - 正文写清：**评哪个界面、哪三条、计划怎么评**。

### Needfinding Results（~2 页 + 附录）

- 用户类活动：**计划 vs 实际**、样本量、高层结果。  
- 启发式：界面在每条启发式上 **符合 / 违背** 的发现。  
- **整体至少 3 条 insight**：应是 needfinding **之前不能确定**、且能用 **具体结果** 论证的观察。  
- 附录：**原始或汇总数据**（图表）、访谈/观察笔记等。

### Initial Brainstorming Plan（~0.5 页）

- 符合课程里 **高效头脑风暴** 的做法；个人项目以 **个人** 为主（可与学习小组互助）。  
- 可用 **AI 辅助**；若用，**附录放对话 transcript**。  
- 说明 **如何减轻头脑风暴常见偏见**。

### Brainstorming Results（~1.5 页 + 附录）

- 总结过程 + 产出的想法；附录可放原始笔记。  
- **至少选 3 个** 设计方向进入原型阶段，并 **简要说明为何选它们**。  
- 每个方向 **约一段文字原型（textual prototype）**。

### Initial Prototyping（~3 页 + 附录）

- **三个** 低保真方案，各 **至少一张图** + 设计 rationale（联系课程概念）。  
- 附录可放更完整屏摄/脚本等。

### Evaluation Planning（~1 页 + 附录）

- **必须有参与者**；比较 **三个原型彼此**，或 **三者 + 现有界面**。  
- 写清：**流程、谁参加、如何招募/激励、具体做什么**（问卷、出声思考、预览原型等）。  
- **必须同时包含**：  
  - **定量**（不必“客观仪器”，可以是李克特偏好、步数、错误次数等可数字化的东西）；  
  - **定性**。  
- **分析计划**：定量打算算什么 / 什么检验；定性如何归纳。  
- 课程说明：**小样本科别指望显著性**，但要 **按计划做分析**。

### Evaluation Results（~2 页 + 附录）

- **执行**评估计划并报告：人数、招募方式等。  
- 按计划在正文中做 **定量分析**（含 p 值等若适用）与 **定性归纳**。  
- 附录：**原始数据**（问卷导出、访谈笔记等）。

### Second Iteration Planning（~2 页）

- 相当于 **再介绍（re-introduction）**：第一轮评估往往 **兼作** 第二轮的 needfinding。  
- 是否还要 **额外** needfinding；设计上的取舍；**最终原型** 是单方案增强还是 **多方案合并**。

### Final Prototype（~3 页 + 附录）

- **中保真**；可基于一轮中某方案加深，或 **组合** 多方案。  
- 正文 **多图** + 用课程概念 **论证关键设计决策**。  
- 更全屏摄可附录；课程称 **视频原型** 常可承担展示职责。

### Video Prototype（~0.5 页 + 外链）

- **演示原型操作** 的视频（非只讲 PPT）；可少剪辑。  
- 约 **5 分钟** 量级 + **本人配音** 走完整流程；穿插 **为何这样设计** 的简要说明。  
- 上传到指定/自选平台，PDF 中给 **链接与访问说明**；**务必用无痕模式自测权限**（GT 账号可访问等）。

### Final Evaluation Planning（~1 页 + 附录）

- 针对 **终版**（视频和/或可交互原型）；**必须有参与者**；**定性 + 定量**。  
- 常见形式：看视频/玩 demo + 少量题（开放 + 量化）。  
- **样本期望**（课程原文）：  
  - **问卷**：约 **20** 份回复；  
  - **访谈或更实时方式**：约 **5** 人。  
- 每人全流程约 **10 分钟** 量级。  
- 写清：**流程、招募、具体问题/变量、定量与定性各自如何分析**（可与基线界面比较，也可能只有描述统计）。

### Final Evaluation Results（~1.5 页 + 附录）

- 执行并分析；附录放 **原始数据**。  
- 定量：按你计划做检验或描述统计并解释。  
- 定性：归纳主题并解释。

## 5. 与你当前方案的对应检查（基于仓库内 `toDo.md`、评估材料）

### 已高度契合的部分

- **任务粒度**：改进 **AP Classroom 类练习/作业反馈**（非整站），符合「具体任务 + 已有界面改进」。  
- **Needfinding 双轨**：计划中有 **访谈** + **启发式评估**，与作业 **两类 needfinding** 一致。  
- **三个方向 → 低保真 → 评估**：A/B/C 与 **n=4 纸面比较** 对应 **三个原型 + Evaluation Planning/Results**；课程要求 **定量+定性**，你的量表+开放题可满足。  
- **第二轮**：中保真 `index.html`、视频、**Final Evaluation Plan / Survey / Interview Guide** 与 **终评** 结构对齐。  
- **附录**：问卷、访谈脚本、笔记、截图、原始数据——你清单里已有规划。

### 需要你在正文里「写满、写清楚」的风险点

| 要求 | 你的现状 / 建议 |
|------|-----------------|
| Needfinding 用户活动样本量 | 访谈路径需 **明确人数与时长**（课程建议 **~5×~20min**）；若问卷也要写清 **n 与是否达到 ~20 的量级**。 |
| Needfinding **≥3 条 insight** | 必须 **带证据**（引访谈原话或启发式对应界面点），避免纯主观。 |
| 启发式评估 | 正文需 **固定界面**（如当前 AP Classroom 学生端反馈页）+ **≥3 启发式** + 每条 **支持/违反** + 与学生目标的关系。 |
| 第一轮 Evaluation | 正文需 **按计划跑统计/描述**（即使不显著），并 **归纳质性主题**；附录 **原始表**。 |
| **终评样本** | 课程写的是 **问卷 ~20** **或** **访谈等 ~5**（约 10 分钟/人）。你目前 **问卷 n=3** 偏少；**用访谈补到约 5 人**（见 `Final_Evaluation_Interview_Guide.md`）是合理路径，但正文要 **分开报告 n、说明局限**，并尽量让 **合并后的证据** 能支撑结论。若仍低于 5，需在 **Limitations** 中诚实说明。 |
| JDF / 页数 | 按节建议加总 **&lt; 25 页** 是给你留余地；图表会占页，需预留。 |
| 外链可访问性 | GitHub Pages、Google Form、视频托管需在 PDF 中写清，并 **自测无痕**。 |

### 总体结论

- **方案结构符合** 课程设计流程与章节划分；主要差距在 **文档化与执行**：把已有计划 **写进对应章节**、附录 **贴齐证据**，并把 **终评样本** 尽量推向课程给出的 **~5（访谈）或 ~20（问卷）** 量级。  
- **团队项目** 记得另选 **与个人项目不延续** 的课题（课程硬性要求）。
